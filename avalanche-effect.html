Avalanche Effect on SHA-256

A key property of a hash function is that the output is unpredictable and even tiny, tiny changes in the output create a completely different output.

This is a simple and visual example of this avalanche effect. The SHA-256 has an internal loop that iterates 64 for times for each 64 bytes of input.

This page shows the internal 32-byte state at each step for two calls to SHA-256:

  1. SHA256('SHA-256'). The input, in hex, is 5348412d323536
  2. SHA256('SHA=256'). The input, in hex, is 5348413d323536. The char = differs from - by only one bit!

Outputs

  1. bbd07c4fc02c99b97124febf42c7b63b5011c0df28d409fbb486b5a9d2e615ea
  2. 029d832a4c29c5b74de7771729e7e06fc9a7438a3c47d6dc7b51441af5809642

Pretty different. See how the output is generated by looking at each intermediate internal state bellow.

The algorithm always starts with state 6a09e667bb67ae853c6ef372a54ff53a510e527f9b05688c 1f83d9ab5be0cd19.
The final state is then added to the initial one to give the final output.

Internal States

For each pair, the first line is the internal state for SHA256('SHA-256') and the second line for SHA256('SHA=256')

At the end, the W vector is shown. The hash input is saved in the first 16 elements of this. All following elements are derived from previous values (as denoted in each one).

W[i] is used in the ith step.

You should note that the avalanche effect is in fact very fast, on the 5th step the single bit change has already affected most of the state bits.
